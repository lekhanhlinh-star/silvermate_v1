# SilverMate API Reference

This document outlines the REST API endpoints available in the SilverMate backend.

## Base URL
`https://implicatively-unpale-susann.ngrok-free.dev`

## Authentication

Authentication is handled via JWT tokens. Include the token in the `Authorization` header as `Bearer <token>` for endpoints that require authentication.

### `POST /auth/register`
Register a new user.
- **Body:**
  ```json
  {
    "email": "user@example.com",
    "password": "securepassword",
    "is_active": true,
    "is_superuser": false,
    "is_verified": false,
    "full_name": "John Doe",
    "user_type": "PARENT" // or "CHILD"
  }
  ```
- **Response:** `201 Created` with user details.

### `POST /auth/jwt/login`
Authenticate a user and receive a JWT token.
- **Content-Type:** `application/x-www-form-urlencoded`
- **Body:**
  - `username`: User's email
  - `password`: User's password
- **Response:** `200 OK`
  ```json
  {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5...",
    "token_type": "bearer"
  }
  ```

### `GET /auth/google/authorize`
Get the Google OAuth authorization URL to redirect the user to Google for login/registration.
- **Response:** `200 OK`
  ```json
  {
    "authorization_url": "https://accounts.google.com/o/oauth2/v2/auth?..."
  }
  ```
*(Note: The frontend should redirect the user to this `authorization_url`)*

### `GET /auth/google/callback`
The callback URL Google redirects back to after successful authentication. If the user doesn't exist, a new account is automatically created.
- **Query Params:** `code`, `state` (provided automatically by Google)
- **Response:** Returns an authentication token (JWT) upon success.

## Connections (Parent-Child Linking)

### `POST /connections/generate-pin`
Generate a temporary 6-digit PIN to link accounts. **Anyone can generate PIN for any connection type.**
- **Requires Auth:** Yes
- **Body:**
  ```json
  {
    "connection_type": "CHILD_TO_PARENT" // Optional: "CHILD_TO_PARENT" or "PARENT_TO_CHILD"
    // If omitted, auto-detects based on user type (Parent→PARENT_TO_CHILD, Child→CHILD_TO_PARENT)
  }
  ```
- **Response:** `200 OK`
  ```json
  {
    "success": true,
    "pin_code": "123456",
    "expires_at": "2023-10-27T10:00:00Z",
    "connection_type": "CHILD_TO_PARENT", // or null if auto-detected
    "message": "PIN generated successfully..."
  }
  ```

### `POST /connections/verify-pin`
Verify a PIN to establish a family connection. **Anyone can verify any PIN - system automatically determines parent-child relationship.**
- **Requires Auth:** Yes
- **Body:**
  ```json
  {
    "pin_code": "123456"
  }
  ```
- **Response:** `200 OK`
  ```json
  {
    "success": true,
    "message": "Family connection created successfully with John Doe",
    "connection_created": true,
    "connected_user": {
        "id": "uuid",
        "email": "user@example.com",
        "full_name": "John Doe",
        "user_type": "PARENT"
    }
  }
  ```
- **Errors:**
  - `400`: Cannot connect two users of the same type (parent-parent or child-child)
  - `400`: Already connected to this family member
  - `400`: Cannot use your own PIN
  - `404`: Invalid PIN or PIN has expired

### `GET /connections/family`
Get all connected family members (children for parents, parents for children).
- **Requires Auth:** Yes
- **Response:** `200 OK` 
  ```json
  [
    {
        "id": "uuid",
        "email": "user@example.com",
        "full_name": "John Doe",
        "user_type": "PARENT"
    }
  ]
  ```

### `GET /connections/my-pins`
Get a list of connection PINs generated by the current user.
- **Requires Auth:** Yes
- **Query Params:** `include_expired` (boolean, default false)
- **Response:** `200 OK` (List of PIN objects)

## Chatbot (For Parents)

### `POST /chatbot/`
Send a message to the SilverMate AI assistant. Streams back the response.
- **Requires Auth:** Yes (Must be a PARENT)
- **Content-Type:** `application/json`
- **Body:**
  ```json
  {
    "message": "Hello SilverMate, how are you today?",
    "session_id": "optional-uuid-here" // If omitted, creates a new session or uses the latest active one
  }
  ```
- **Response:** `200 OK` Chunked streaming plain text from the AI.

## Monitoring (For Children)

### `GET /monitoring/{parent_id}/sessions`
List all chat sessions for a specific connected parent.
- **Requires Auth:** Yes (Must be a CHILD connected to the `parent_id`)
- **Response:** `200 OK`
  ```json
  [
    {
      "id": "uuid",
      "user_id": "parent-uuid",
      "created_at": "2023-10-27T10:00:00Z",
      "updated_at": "2023-10-27T10:05:00Z",
      "summary": "Summary of the chat...",
      "is_active": true
    }
  ]
  ```

### `GET /monitoring/{parent_id}/sessions/{session_id}/messages`
Get the raw messaging history for a specific session, including audio recordings if available.
- **Requires Auth:** Yes
- **Response:** `200 OK`
  ```json
  [
    {
      "id": "uuid",
      "session_id": "session-uuid",
      "role": "user",
      "content": "Hello",
      "created_at": "2023-10-27T10:00:00Z",
      "audio_url": "https://example.com/audio/message-abc123.webm", // Optional: URL to audio recording
      "is_voice_message": true // Optional: Indicates if this was originally a voice message
    },
    {
      "id": "uuid",
      "session_id": "session-uuid",
      "role": "assistant",
      "content": "Hi there! How can I help you today?",
      "created_at": "2023-10-27T10:00:05Z"
    }
  ]
  ```
- **Note:** Messages with `audio_url` provide the original audio recording from the parent. The `content` field contains the transcribed text. The `is_voice_message` flag helps identify voice messages even when audio URLs are not available (e.g., for older messages or when audio storage is disabled).

### `POST /monitoring/{parent_id}/sessions/{session_id}/summarize`
Trigger the AI to analyze and summarize a specific chat session.
- **Requires Auth:** Yes
- **Response:** `200 OK`
  ```json
  {
    "summary": "The parent discussed their garden and felt very happy today. They mentioned wanting to call you later."
  }
  ```
